# 修仙 Roguelike 游戏设计文档

## 1. 游戏概述

这是一个纯前端的 AI 驱动修仙 Roguelike 游戏，所有游戏内容（角色、世界观、剧情、NPC、物品等）都由大模型实时生成。玩家通过自填 API Key，api 地址，模型等参数接入大模型，体验独一无二的修仙之旅。

### 核心特色

* **纯前端架构**：无后端，所有数据存储在浏览器本地

* **AI 完全驱动**：世界观、角色、剧情、NPC 全部由 LLM 生成

* **高自由度**：无固定主线，玩家想做什么就做什么

* **无限随机性**：每次游玩都是全新体验

* **本地存档**：支持保存和加载游戏进度

***

## 2. 游戏流程

### 2.1 角色创建阶段

1. 玩家点击"踏入仙途"
2. AI 随机生成 3 个不同的角色，每个角色包含：

   * **身世**：随机出生背景（例如：农家子弟、孤儿、世家公子等）

   * **天赋**：3-5 个修仙天赋（例如：先天道体、过目不忘、五行俱全等）

   * **基础属性**：气血、真气、攻击、防御、速度、气运、根骨、悟性

   * **外观**：Emoji 组合（例如：🧑‍🦰👕🔮）
3. 玩家选择其中一个角色开始游戏

### 2.2 游戏主阶段

1. 大模型提供玩家目前状态下可以行动的选项，也可以在输入框中输入行动指令（例如："去后山修炼"、"探索洞穴"、"与老者交谈"）
2. AI 作为 DM（地下城主），结合以下信息推演出剧情：

   * 当前角色属性、技能、性格

   * 人际关系

   * 最近的历史事件

   * 游戏世界观设定
3. AI 自动判定：

   * 是否获得奇遇

   * 是否获得灵气/修为增长

   * 是否突破境界

   * 是否获得新物品/功法

   * 是否结识新 NPC

   * 消耗了多少寿元/时间
4. 剧情和结果展示在右侧日志中
5. 循环上述流程

## 3. 游戏系统设计

### 3.1 角色系统

#### 属性定义

| 属性 | 说明                  |
| -- | ------------------- |
| 气血 | 生命值，归零时死亡           |
| 真气 | 施法/技能消耗             |
| 攻击 | 物理/法术伤害             |
| 防御 | 减少受到的伤害             |
| 速度 | 决定行动顺序              |
| 气运 | 影响奇遇概率、掉落率          |
| 根骨 | 影响修炼速度、突破成功率、气血上限   |
| 悟性 | 影响领悟功法、技能学习速度、突破成功率 |
| 修为 | 当前境界的修为进度（0%-100%）  |
| 灵气 | 可用于提升修为的灵气储备        |
| 寿元 | 剩余寿命（年），归零时死亡       |
| 年龄 | 当前年龄                |

#### 境界体系

参照主流修仙小说设定，采用经典九境体系，每个大境界分为初期、中期、后期、巅峰四个小境界：

1. **炼气期**（凡人修仙之始，吸收天地灵气）

   * 炼气一层 → 炼气二层 → ... → 炼气十层

   * 或简化为：炼气初期 → 炼气中期 → 炼气后期 → 炼气巅峰
2. **筑基期**（奠定道基，寿元增至 200-300 岁）

   * 筑基初期 → 筑基中期 → 筑基后期 → 筑基巅峰
3. **金丹期**（修成金丹，寿元增至 500 岁）

   * 金丹初期 → 金丹中期 → 金丹后期 → 金丹巅峰
4. **元婴期**（破丹成婴，寿元增至 1000 岁）

   * 元婴初期 → 元婴中期 → 元婴后期 → 元婴巅峰
5. **化神期**（元婴化神，寿元增至 2000 岁）

   * 化神初期 → 化神中期 → 化神后期 → 化神巅峰
6. **炼虚期**（炼虚合道，寿元增至 5000 岁）

   * 炼虚初期 → 炼虚中期 → 炼虚后期 → 炼虚巅峰
7. **合体期**（天人合一，寿元增至 10000 岁）

   * 合体初期 → 合体中期 → 合体后期 → 合体巅峰
8. **大乘期**（大乘可期，寿元增至 50000 岁）

   * 大乘初期 → 大乘中期 → 大乘后期 → 大乘巅峰
9. **渡劫期**（渡劫飞升，寿元无限）

   * 渡劫初期 → 渡劫中期 → 渡劫后期 → 渡劫巅峰

   * 渡过天劫 → **飞升仙界**

#### 修为进度系统

* 每个境界都有独立的修为进度（0% - 100%）

* 修为达到 100% 时，可以尝试突破到下一境界

* 修为增长方式：

  * 修炼/打坐：消耗时间，获得修为

  * 服用丹药：快速获得修为，可能有丹毒风险

  * 吸收天材地宝：获得大量修为

  * 奇遇/传承：获得修为

  * 战斗胜利：获得少量修为

* 突破机制：

  * 修为 100% 后可选择突破

  * 突破有成功率，受根骨、悟性、气运、心境、丹药辅助影响

  * 突破失败可能导致修为倒退、心魔、甚至死亡

  * 突破成功后，修为清零，进入下一境界

#### 灵气系统

* 灵气是修炼的基础资源

* 灵气获取方式：

  * 在灵气浓郁的地方修炼/打坐

  * 服用丹药

  * 吸收灵石

  * 吸收天材地宝

  * 某些特殊功法

* 灵气用途：

  * 转化为修为

  * 释放法术/技能

  * 疗伤

  * 激活法阵

  * 祭炼法宝

#### 寿元/时间系统

* 每个角色有初始寿元，根据天赋、种族、境界变化

* 初始寿元参考：

  * 凡人：60-100 岁

  * 炼气期：100-150 岁

  * 筑基期：200-300 岁

  * 金丹期：500 岁

  * 元婴期：1000 岁

  * 化神期：2000 岁

  * ...以此类推

* 时间消耗机制：

  * 所有行动都会消耗时间（寿元）

  * 常见行动时间消耗示例：

    * 短途移动（城内）：1-6 时辰

    * 长途移动（跨城）：1-30 天

    * 普通修炼：1-7 天

    * 闭关修炼：1-12 个月

    * 探索秘境：1-30 天

    * 炼制丹药/法器：1-30 天

    * 学习功法：1-30 天

    * 战斗：数时辰

* 时间单位：

  * 1 时辰 = 2 小时

  * 1 天 = 12 时辰

  * 1 月 = 30 天

  * 1 年 = 12 月

* 特殊时间事件：

  * 月圆之夜：某些功法效果增强，某些妖兽出没

  * 百年一遇的天劫：大乘期后触发

  * 灵气潮汐：灵气浓度大幅提升

  * 秘境开启：限时开放

#### 天赋系统

* 每个角色初始拥有 3-5 个天赋

* 天赋影响属性成长、技能学习、奇遇概率等

* 示例天赋：

  * **先天道体**：修炼速度 +100%，突破成功率 +50%

  * **五行俱全**：可修炼任何属性功法，无属性克制

  * **过目不忘**：学习功法速度 +200%

  * **天命之子**：奇遇概率 +300%，气运永久 +50

  * **厄运缠身**：奇遇概率 -50%，但每次化险为夷后获得大量奖励

  * **先天残缺**：某项属性极低，但另一项极高

  * **寿元绵长**：初始寿元 +200%

  * **丹毒不侵**：服用丹药无丹毒风险

  * **天生媚骨**：异性 NPC 好感度初始 +30

  * **剑骨**：剑法类功法修炼速度 +150%

## 5. 数据持久化

### 5.1 存储方案

采用 **localStorage + IndexedDB 混合存储方案**：

| 存储方式 | 适用场景 | 容量限制 |
|--------|--------|--------|
| **localStorage** | 游戏设置、当前存档元数据、用户偏好 | ~5MB |
| **IndexedDB** | 完整游戏存档（剧情日志、NPC 历史、背包等） | 250MB+（甚至无上限） |

#### 存储分工

* **localStorage** 存储：
  * LLM 配置（API Key、Base URL、模型名称）
  * UI 主题设置
  * 当前活跃存档 ID
  * 存档列表元数据（存档名称、时间、角色等级等摘要信息）

* **IndexedDB** 存储：
  * 完整存档数据（角色、NPC、物品、剧情日志等）
  * 历史消息/记忆向量（用于 RAG 检索）

### 5.2 IndexedDB 数据结构

```typescript
// IndexedDB Object Stores
- saves: 存档列表
  { id, name, timestamp, playerLevel, realm, summary }

- saveData: 完整存档数据
  { saveId, data: { player, npcs, inventory, logs, world, ... } }

- memories: 记忆片段（用于 RAG）
  { id, saveId, type, content, embedding, timestamp, importance }
```

### 5.3 存档功能

* 自动保存（每回合/定时）

* 手动保存

* 存档列表管理

* 存档导出/导入（JSON）

## 6. LLM 集成与记忆管理

### 6.1 支持的 API

* OpenAI 兼容格式

* 支持自定义 Base URL

* 支持自定义模型名称

### 6.2 提示词设计

将设计以下提示词：

1. **角色生成提示词**：生成 3 个随机角色
2. **剧情推演提示词**：根据玩家行动推演剧情
3. **世界生成提示词**：生成初始世界观
4. **NPC 生成提示词**：生成 NPC
5. **物品生成提示词**：生成物品/功法
6. **记忆摘要提示词**：将长历史压缩为摘要

### 6.3 多层记忆系统（优化 Token 成本）

为解决记忆不断增长导致 Token 消耗过高的问题，采用**多层记忆架构**：

```
┌─────────────────────────────────────────────────────────┐
│  第一层：工作记忆（最新 N 条对话）                        │
│  - 完整保留最近 10-20 条对话                              │
│  - 直接放入上下文窗口                                      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第二层：摘要记忆（历史摘要）                               │
│  - 更早的对话压缩成摘要（每 50 条摘要一次）                 │
│  - 使用 LLM 生成结构化摘要（关键事件、人物关系变化等）       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第三层：检索记忆（RAG）                                  │
│  - 所有记忆片段向量化存储在 IndexedDB                     │
│  - 根据当前查询语义检索 Top-K 相关记忆                     │
│  - 仅将相关片段放入上下文，大幅节省 Token                   │
└─────────────────────────────────────────────────────────┘
```

#### 记忆片段类型

| 类型 | 说明 | 重要性权重 |
|-----|------|----------|
| 角色状态变化 | 突破境界、获得重要物品、寿元变化 | 高 |
| NPC 交互 | 结识新 NPC、好感度大幅变化 | 高 |
| 关键事件 | 获得奇遇、触发剧情转折点 | 高 |
| 普通对话 | 日常闲聊、探索 | 中 |
| 背景信息 | 世界观介绍、场景描述 | 低 |

#### 记忆压缩与摘要策略

1. **滑动窗口**：保留最新 K 条完整对话
2. **定期摘要**：当历史超过阈值时，调用 LLM 将旧对话压缩为摘要
3. **重要性过滤**：优先保留高重要性记忆，低重要性记忆可更早被压缩
4. **向量检索**：使用轻量级嵌入模型（如 all-MiniLM-L6-v2）在浏览器本地生成向量，根据当前查询检索相关记忆

#### Token 优化效果

通过多层记忆架构，预期可将 Token 消耗降低 **70%-90%**，同时保持对话连贯性和上下文感知能力。

### 6.4 提示词上下文组装

每次调用 LLM 时，按以下顺序组装上下文：

1. **系统提示词**（固定）
2. **当前角色状态**（属性、修为、寿元、背包等）
3. **摘要记忆**（历史压缩摘要）
4. **检索记忆**（Top-K 相关片段）
5. **工作记忆**（最近 N 条完整对话）
6. **当前用户输入**

## 7. 游戏化设计要点

### 7.1 核心循环

```
行动 → 消耗时间 → 获得反馈（修为/物品/剧情） → 成长 → 继续行动
```

### 7.2 风险与回报

* 修炼越久，修为越多，但寿元消耗也越多

* 高风险行动（探索绝地、挑战高阶修士）可能获得高回报

* 丹药效果显著，但有丹毒风险

* 突破有失败风险，但成功后实力大幅提升

### 7.3 短期与长期目标

* 短期目标：提升修为、收集资源、完成当前事件

* 长期目标：突破境界、延长寿元、建立势力、飞升成仙

### 7.4 随机要素

* 角色身世、天赋随机

* 遭遇的 NPC、事件随机

* 物品掉落、奇遇随机

* 突破成功率随机
