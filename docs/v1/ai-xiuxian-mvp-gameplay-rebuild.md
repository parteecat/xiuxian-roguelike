# AI 修仙 Roguelike 游戏 MVP 玩法重构文档

## 1. 文档目标

本次重构目标不是继续强化“小说生成”，而是把当前项目调整为：

**一个基于文字 + 简单图标 + 纯前端 + 大模型驱动的修仙生存构筑 Roguelike。**

重点解决当前版本的问题：

- 更像互动小说，不像游戏
- 玩家输入自由度高，但决策感弱
- 成长有数值，但缺少流派构筑乐趣
- 事件很多，但缺少风险、压力、代价
- 玩一会后缺少“再来一回合”的上瘾感

## 2. 核心设计原则

### 2.1 AI 的职责调整

AI 不再主要承担“续写故事”的职责，而是承担以下职责：

1. 根据当前状态生成合理事件
2. 根据规则输出结构化结果
3. 根据玩家构筑方向投喂对应机缘
4. 根据世界状态推进连锁反应
5. 在规则边界内补充文本表现力

### 2.2 游戏优先于叙事

每一回合优先输出：

- 当前局势
- 风险提示
- 资源变化
- 可选策略
- 结果反馈

剧情文本只作为包装层，不再作为主体。

### 2.3 默认选项驱动，自由输入为补充

主要交互改为：

- 系统提供 3~5 个高质量策略选项
- 玩家点击选择
- 保留“自定义行动”按钮作为高级玩法入口

## 3. 核心玩法定位

一句话定义：

> 玩家在一个持续演化的修仙世界里，通过资源管理、风险选择、地图探索、流派构筑与连锁事件，尽可能活得更久、修得更高、拿到更强机缘。

## 4. MVP 核心循环

### 4.1 单回合循环

每回合流程：

1. 结算世界状态变化
2. 展示当前地点信息
3. 展示角色资源状态
4. 展示当前风险/机会提示
5. 生成 3~5 个策略型选项
6. 玩家做出选择
7. AI 按规则结算结果
8. 更新数值 / 状态 / 地图 / NPC / 时间
9. 进入下一回合

### 4.2 中期循环

玩家在多个回合内完成：

- 收集资源
- 修炼提升
- 探索地图
- 应对危机
- 参与事件
- 构筑流派
- 决定是否突破境界

### 4.3 长期循环

一次游戏生命周期包含：

- 开局天赋 / 身份选择
- 炼气 → 筑基 → 金丹...
- 结识 NPC / 树敌 / 结缘
- 获取机缘 / 传承 / 遗迹
- 达成结局 / 死亡
- 解锁局外内容
- 开启下一局

## 5. MVP 必做系统

## 5.1 角色核心资源系统

MVP 必须突出展示以下资源：

### 基础资源

#### 1）气血
- 代表生命状态
- 战斗、探索失败、强行突破会损失
- 过低时很多行动失败率上升

#### 2）真气
- 代表法术 / 战斗 / 修炼消耗
- 某些流派会高频消耗真气
- 真气不足时无法发动强力行为

#### 3）修为
- 当前境界成长进度
- 例如：炼气三层 67%
- 达到阈值后可尝试突破

#### 4）寿元
- 长期压力资源
- 某些禁术、重伤、邪法会消耗寿元
- 寿元越低，某些事件风险越高

#### 5）灵石
- 经济资源
- 用于购买丹药、情报、功法、补给、传送等

### 扩展资源

#### 6）心魔值
- 高风险行为、失败突破、杀戮、负面事件会增加
- 影响突破成功率
- 过高时触发特殊负面事件

#### 7）声望 / 恶名
- 影响 NPC 态度、宗门态度、事件分支
- 正道路线主要吃声望
- 邪道路线主要吃恶名

## 5.2 时间与回合压力系统

### 时间单位
- 推荐使用“天”作为最小时间单位
- 每次行动消耗 1~3 天不等

### 时间系统用途

时间不是背景描述，而是压力来源：

- 坊市每 7 天刷新
- 秘境每 30 天开启
- 宗门收徒每 60 天一次
- 仇家追踪会在若干天后到来
- 某些 NPC 事件带倒计时
- 玩家年龄持续增长，寿元持续消耗

### 设计目标

让玩家经常面临：

- 现在去疗伤，还是去赶秘境
- 现在闭关，还是先赚钱买丹
- 现在突破，还是继续稳一点

## 5.3 结构化回合输出系统

每一回合必须固定输出如下结构：

```md
### 当前地点
青木坊市 · 东街

### 当前状态
- 境界：炼气四层（45%）
- 气血：72/100
- 真气：31/60
- 寿元：43/60 年
- 灵石：28
- 心魔：12
- 声望：4
- 恶名：1

### 当前局势
你刚从黑风岭归来，身上带着一株受损灵草。坊市里似乎有人在盯着你。
三日后，黑市拍卖会开启。

### 风险提示
- 气血偏低，继续高风险探索容易重伤
- 你携带灵草，可能触发觊觎事件
- 黑市拍卖将在 3 天后开启

### 可选行动
1. 去药铺疗伤（稳妥，花费灵石）
2. 去黑市打听灵草价格（中收益，中风险）
3. 直接返回客栈闭关炼化灵草（高收益，中风险）
4. 反向追踪盯梢者（高风险，高收益）
5. 自定义行动
```

## 5.4 地图与地点玩法系统

地图不能只是背景，需要变成策略盘。

### MVP 地图结构建议

至少包含以下地点类型：

#### 1）安全区
如：
- 宗门
- 坊市
- 客栈
- 洞府

特点：
- 恢复资源
- 交易
- 打听消息
- 接任务
- 风险低

#### 2）资源区
如：
- 药田
- 山林
- 湖泊
- 矿洞

特点：
- 获取基础资源
- 风险适中
- 适合前期积累

#### 3）危险区
如：
- 妖兽山脉
- 邪修洞府
- 乱葬岗
- 禁地外围

特点：
- 高收益
- 高风险
- 可能触发敌对事件 / Boss / 机缘

#### 4）特殊区
如：
- 秘境入口
- 上古遗迹
- 黑市拍卖
- 雷池
- 情缘地点

特点：
- 需要条件开启
- 事件导向强
- 产出稀有内容

### 地点需要提供的信息

每个地点至少要有：

- 名称
- 类型
- 风险等级
- 推荐境界
- 可行动列表
- 掉落 / 收益倾向
- 特殊状态

## 5.5 行动系统

### 行动必须是“策略型选项”

错误示例：
- 去森林看看
- 去森林探索
- 去森林逛逛

正确示例：
- 稳妥采药（低风险，低收益）
- 深入狩猎（中风险，中收益）
- 追踪妖气源头（高风险，高收益）
- 绕路侦查地形（低收益，获取情报）
- 设伏截杀他人（邪道行为，高收益高代价）

### 每个行动应包含以下标签

- 风险等级
- 收益类型
- 时间消耗
- 资源消耗
- 特殊倾向标签

例如：

```ts
type ActionOption = {
  id: string
  title: string
  desc: string
  risk: 'low' | 'mid' | 'high'
  rewardType: string[]
  cost: {
    hp?: number
    mp?: number
    spiritStone?: number
    time?: number
  }
  tags: string[]
}
```

## 5.6 风险与概率系统

这是把“小说”变成“游戏”的关键。

### 核心要求

关键行为必须尽量可见概率，而不是完全黑箱。

例如：

#### 突破
- 成功率：58%
- 轻伤率：20%
- 重伤率：12%
- 走火入魔率：8%
- 死亡率：2%

#### 探索禁地
- 获得机缘率：30%
- 获得普通材料率：40%
- 遭遇敌人率：20%
- 重伤率：8%
- 死亡率：2%

### 概率受以下因素影响

- 当前境界
- 地点环境
- 功法标签
- 角色天赋
- 心魔值
- 当前资源状态
- 装备 / 丹药 / NPC 帮助

### 设计原则

玩家需要知道：
- 我在赌什么
- 这个赌值不值
- 我如何提高成功率

## 5.7 流派构筑系统

这是复玩性的核心。

### MVP 流派方向

先做 6 类标签流派：

#### 1）剑修
特点：
- 高爆发
- 单体强
- 比较脆
- 适合速战

#### 2）体修
特点：
- 高气血
- 高容错
- 恢复强
- 成长偏慢

#### 3）法修
特点：
- 高频消耗真气
- 法术范围强
- 适合控场和爆发

#### 4）丹修
特点：
- 强资源转化
- 能炼丹 / 解毒 / 增益
- 前期偏弱，后期稳定

#### 5）魅修 / 社交流
特点：
- 擅长 NPC 互动
- 情报、交易、结缘收益高
- 可降低敌意事件概率

#### 6）邪修
特点：
- 高风险高收益
- 容易获得捷径
- 心魔、恶名、副作用高

### 流派实现方式

不需要做复杂职业系统，而是使用“标签累计”：

例如角色面板：

```md
### 构筑倾向
- 剑修：3
- 体修：1
- 法修：0
- 丹修：2
- 魅修：0
- 邪修：1
```

### 标签来源

以下内容都可以给标签：

- 初始天赋
- 功法
- 法宝
- 关键事件
- NPC 师承
- 选择行为

### 标签作用

达到阈值时：
- 解锁对应专属行动
- 解锁专属事件
- 增强某类收益
- 改变突破方式
- 改变结局倾向

## 5.8 战斗系统（简化版）

不做复杂战斗面板，但不能只是一段文本判定。

### MVP 战斗改成三段式

#### 第一阶段：开局策略
可选：
- 先手强攻
- 试探观察
- 设法周旋
- 立刻逃离

#### 第二阶段：中盘决策
可选：
- 爆发一击
- 防守回气
- 使用丹药
- 呼叫援手
- 施展底牌

#### 第三阶段：残局选择
可选：
- 斩杀
- 留手
- 夺宝
- 逼问情报
- 趁机撤退

### 这样做的好处

- 保留文字游戏的轻量化
- 增加战斗中的决策感
- 能制造不同后果
- 不需要额外复杂战斗 UI

## 5.9 NPC 系统重构

NPC 不能只是“聊天对象”，必须是玩法节点。

### 每个关键 NPC 至少具备以下功能中的 2~3 个

- 交易
- 任务发布
- 情报提供
- 师承 / 传功
- 结缘 / 情感线
- 背叛 / 算计
- 阵营关系变化
- 特殊事件触发

### NPC 基础数据结构建议

```ts
type Npc = {
  id: string
  name: string
  title: string
  camp: string
  favor: number
  danger: number
  tags: string[]
  secrets: string[]
  functions: ('trade' | 'quest' | 'intel' | 'mentor' | 'romance' | 'enemy')[]
}
```

### NPC 需要具备的变量

- 好感度
- 危险度
- 立场
- 隐藏目的
- 对玩家印象
- 可触发事件链

## 5.10 连锁事件系统

当前很多事件像一次性段子，需要改成“事件链”。

### 事件链例子

#### 例子 1：救人事件链
- 在山林救下一位重伤少女
- 后续发现她是魔宗修士
- 正道宗门开始怀疑你
- 她会在你危机时回报 / 出卖你

#### 例子 2：捡漏事件链
- 在坊市低价买到残卷
- 被人盯上
- 黑市发布悬赏
- 你可选择卖掉 / 修炼 / 设局反杀

### 事件链必备字段

- 当前阶段
- 触发条件
- 剩余时限
- 涉及 NPC / 阵营
- 可能奖励
- 可能代价

## 5.11 突破系统

突破必须成为大节点，而不是简单数值升级。

### 突破前应展示

- 当前成功率
- 风险概率
- 影响因素
- 可使用辅助资源

### 突破方式

#### 1）稳妥突破
- 成功率高
- 消耗资源高
- 收益正常

#### 2）强行突破
- 成功率低
- 风险高
- 可能直接跨级获得特殊词条

#### 3）借物突破
- 消耗丹药 / 法宝 / 阵法 / NPC 帮助
- 需要前置准备

#### 4）邪法突破
- 收益高
- 有明显副作用
- 增加心魔 / 恶名 / 寿元损耗

## 6. 局外成长系统（Meta）

MVP 就算做轻量，也必须有。

### 玩家死亡 / 结局后给予以下回报之一

- 解锁新出身
- 解锁新初始天赋
- 解锁新流派相关事件
- 解锁新地图区域
- 解锁新世界词缀
- 解锁前世遗物

### 前世遗物例子

- 断裂飞剑：开局剑修倾向 +1
- 枯木丹炉：开局丹修倾向 +1
- 染血玉佩：开局恶名 +1，但可触发黑市事件
- 残页心法：开局修为 +5%

### 作用

让失败也有正反馈，避免“死了白玩”。

## 7. 世界词缀系统

每一局随机给 2~3 个世界修正词缀，提升复玩性。

### 示例词缀

#### 灵气紊乱
- 修炼收益降低
- 奇遇概率提升

#### 正道昌盛
- 安全区更安全
- 邪修路线收益下降

#### 魔潮暗涌
- 危险区收益提升
- 恶性事件变多

#### 丹道盛世
- 丹药价格下降
- 丹毒事件增加

#### 情劫之世
- NPC 情感事件更频繁
- 结缘收益提升
- 情感类负面后果也更强

## 8. 单回合输出规范（给 AI 的规则）

### 输出格式要求

AI 每回合必须输出以下区块：

```md
### 地点
### 状态
### 当前局势
### 风险提示
### 可选行动
### 备注（可选）
```

### 输出限制

- 不允许只输出大段剧情
- 不允许忽略状态变化
- 不允许不给玩家选项
- 每回合选项数控制在 3~5 个
- 每个选项必须有明显差异
- 必须有至少一个低风险选项
- 高风险选项必须附带清晰收益诱因

## 9. 前端 UI 建议

纯前端文字版也可以做出“游戏味”。

### 页面布局建议

#### 左侧：角色与世界面板
显示：
- 境界
- 气血 / 真气 / 修为 / 寿元
- 灵石
- 心魔
- 声望 / 恶名
- 当前地点
- 当前时间
- 当前倒计时事件
- 构筑标签

#### 中间：主叙事区
显示：
- 当前局势摘要
- 事件文本
- 行动结果

#### 右侧：行动区
显示：
- 3~5 个选项按钮
- 自定义行动输入框
- 历史记录折叠面板

### 必须强调展示的内容

- 本回合得失
- 风险提示
- 倒计时
- 构筑方向
- 当前异常状态

## 10. MVP 版本范围控制

为了避免项目继续失控，MVP 只做以下内容：

### 必做
- 基础属性资源系统
- 时间推进
- 地图地点切换
- 结构化行动选项
- 风险概率结算
- 简化战斗三段式
- 基础流派标签
- 基础 NPC 系统
- 基础连锁事件
- 死亡结算 + 局外解锁

### 不做
- 复杂多人关系网
- 超长小说式世界剧情主线
- 过度自由的自然语言解析
- 复杂装备系统
- 复杂阵法 / 炼器 / 宗门管理
- 超细粒度战斗技能树
- 服务端排行 / 联网内容

## 11. 推荐的首个 MVP 场景包

建议先只做以下内容验证游戏性：

### 初始地图
- 青木坊市（安全区）
- 黑风岭（资源/危险区）
- 破庙（特殊事件区）
- 宗门山门（机会区）
- 后山洞府（修炼区）

### 初始 NPC
- 药铺掌柜
- 可疑老者
- 落魄散修
- 宗门执事
- 神秘少女

### 初始流派支持
- 剑修
- 丹修
- 邪修

### 初始事件链
- 残卷奇遇
- 救人事件
- 坊市盯梢事件
- 宗门收徒事件
- 秘境开启事件

## 12. 成功判断标准

如果 MVP 重构成功，玩家应在前 10 分钟内感受到：

1. 我每回合都在做真正有差异的选择
2. 我做错事会付代价
3. 我开始形成自己的修炼路线
4. 世界不是静止的，它在推进
5. 我死了也想再开一局试别的流派

如果玩家玩起来还是：
- 一直在看 AI 讲故事
- 不关心数值
- 不在意地点
- 不在意时间
- 选什么都差不多

那说明重构仍未成功。

## 13. 一句话版本的改造目标
